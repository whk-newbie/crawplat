# E2E 冒烟检查清单

> 用于验证平台核心流程和双语能力的最终回归检查清单。建议在每次合并到 main 前执行。

## 1. 核心业务流程冒烟

### 1.1 认证流程

- [ ] **登录**：使用 `admin` / `admin123` 登录成功，跳转到项目管理页
- [ ] **注册**：新用户注册成功后跳转到登录页
- [ ] **Token 持久化**：刷新页面后仍保持登录状态
- [ ] **登出**：退出登录后无法访问受保护页面

### 1.2 项目管理

- [ ] **创建项目**：填写项目编码和名称，创建成功
- [ ] **列表展示**：项目列表正确显示已创建的项目
- [ ] **空状态**：无项目时显示空状态提示

### 1.3 爬虫管理

- [ ] **创建爬虫**：选择项目和填写信息，创建成功
- [ ] **列表展示**：爬虫列表按项目筛选
- [ ] **版本管理**：创建版本并设为当前版本
- [ ] **仓库凭据引用**：加载和使用 registry auth ref

### 1.4 执行管理

- [ ] **创建执行**：选择项目、爬虫、镜像和命令，创建成功
- [ ] **执行详情**：查看执行状态、节点、触发方式等
- [ ] **执行日志**：日志页面正确展示执行输出
- [ ] **资源限制**：CPU、内存、超时字段在创建和详情页均可正确展示

### 1.5 调度管理

- [ ] **创建调度**：配置 cron 表达式和重试策略，创建成功
- [ ] **列表展示**：调度列表正确显示状态和配置
- [ ] **启停控制**：启用/禁用调度

### 1.6 节点管理

- [ ] **节点列表**：显示在线/离线节点及心跳时间
- [ ] **节点详情**：查看心跳历史和执行历史
- [ ] **会话记录**：查看节点会话聚合视图
- [ ] **状态筛选**：按执行状态过滤历史记录

### 1.7 数据源管理

- [ ] **创建数据源**：选择类型并填写配置，创建成功
- [ ] **连接测试**：test 操作返回正确状态
- [ ] **数据预览**：preview 操作返回数据行

### 1.8 监控概览

- [ ] **概览页面**：统计卡片显示执行和节点计数器
- [ ] **刷新功能**：点击刷新按钮更新数据
- [ ] **原始数据**：原始 JSON 数据区域正确渲染

## 2. 中英双语切换冒烟

### 2.1 语言切换基础

- [ ] **默认语言**：首次访问默认为中文

- [ ] **持久化**：刷新页面后语言选择保持不变
- [ ] **路由保持**：切换语言后当前页面路由不丢失

- [ ] 登录页：标题、表单字段、按钮、错误提示
- [ ] 注册页：标题、表单字段、按钮、成功提示
- [ ] 项目管理：页面标题、表格列头、按钮、弹窗、空状态
- [ ] 爬虫管理：页面标题、表单、版本管理弹窗
- [ ] 执行管理：创建表单、详情页、日志页
- [ ] 调度管理：表格、cron 提示、重试字段说明
- [ ] 节点管理：列表、详情、会话、在线/离线标签
- [ ] 数据源管理：表格、test/preview 结果
- [ ] 监控页：统计卡片标题、原始数据区域

### 2.3 共享组件

- [ ] 导航菜单项在中英文下正确显示
- [ ] 加载状态文案跟随语言变化
- [ ] 空状态文案跟随语言变化

- [ ] Toast 通知文案跟随语言变化

## 3. 典型错误状态冒烟

### 3.1 认证错误

- [ ] **未登录访问**：访问受保护页面显示登录提示
- [ ] **错误密码**：登录失败显示对应语言错误提示
- [ ] **重复注册**：注册已存在用户显示冲突提示

### 3.2 请求错误

- [ ] **404 页面**：访问不存在路由显示 404 错误页
- [ ] **API 404**：请求不存在的资源显示对应语言提示
- [ ] **API 500**：服务异常时显示通用错误提示
- [ ] **网络错误**：网络中断时显示连接错误提示

### 3.3 表单校验错误

- [ ] **必填字段**：提交空表单显示字段校验提示
- [ ] **格式校验**：非法输入显示对应语言提示

## 4. 文档与术语冒烟

### 4.1 文档完整性

- [ ] 核心产品文档存在中英双语版本
- [ ] 核心设计文档存在中英双语版本
- [ ] 架构文档存在中英双语版本
- [ ] 开发指南存在中英双语版本

### 4.2 术语一致性

- [ ] 前端页面中的术语与 `platform-terminology.md` 一致
- [ ] 文档中的术语与前端页面中的术语一致
- [ ] API 错误消息中的术语与前端映射一致

### 4.3 注释规范

- [ ] 新增 Go 文件包含中文文件注释
- [ ] 核心函数包含中文函数注释
- [ ] 注释说明「为什么这样做」而非仅重复「做了什么」

## 执行说明

本清单可在每次合并到 main 前按以下方式执行：

1. 启动完整服务栈：`make migrate && make up`
2. 按清单逐项验证，记录不通过的项
3. 切换语言为英文后重新执行第 2 节
4. 执行 `make down` 清理环境

不通过的项应在合并前修复或在 PR 描述中明确说明。
