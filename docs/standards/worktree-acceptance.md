# Worktree 验收规范

> 适用范围：所有功能、文档、前端、后端、CI、测试相关 worktree。

## 通用最小验收清单

每个 worktree 合并前至少满足：

- [ ] 变更范围集中在该 worktree 的职责边界内。
- [ ] 不依赖未合并的其他临时分支。
- [ ] 相关测试、检查或文档校验已执行。
- [ ] 新增关键逻辑具备必要注释或文档说明。
- [ ] 改动后的契约与设计文档一致，或文档已同步更新。
- [ ] git diff 可明确说明“改了哪些目录、为什么改”。

## 后端 service worktree

至少检查：

- [ ] 目标服务可编译或可通过相关测试。
- [ ] API 路由、状态码、错误响应与设计一致。
- [ ] 关键函数、状态机、中间件具备中文注释。
- [ ] 不顺手修改其他服务的无关逻辑。
- [ ] 如新增稳定错误语义，已同步 `docs/standards/error-messages.md` 或相关设计文档。

## Gateway worktree

至少检查：

- [ ] 统一错误响应仍为 `{ "error": "message" }`。
- [ ] request-id 透传、返回和日志字段正常。
- [ ] JWT / internal token 失败路径有覆盖。
- [ ] 新增路由不存在冲突。
- [ ] 上游不可用时返回稳定错误语义。

## 前端 worktree

至少检查：

- [ ] 中文、英文都可渲染。
- [ ] 语言切换不丢失主要上下文。
- [ ] 不存在明显硬编码中文。
- [ ] 空状态、错误状态、加载状态具备翻译。
- [ ] 英文长度不会破坏主要布局。

## 文档 worktree

至少检查：

- [ ] 文档位于正确目录。
- [ ] 术语遵循平台术语表。
- [ ] 双语文档标题、表格、示例同步。
- [ ] 占位内容明确可识别。
- [ ] 文档中引用的接口、状态码、字段名与代码一致。

## 验证与记录

推荐在 worktree 完成说明中记录：

1. 本次 worktree 名称；
2. 变更目录；
3. 主要契约变化；
4. 运行了哪些验证命令；
5. 是否需要后续 worktree 跟进。
