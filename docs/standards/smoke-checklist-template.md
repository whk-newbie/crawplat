# Smoke / 回归检查模板

> 适用范围：前端、后端、文档、CI 相关 worktree 的最小回归模板。

## 1. 基本信息

- Worktree / 分支：
- 负责人：
- 目标范围：
- 关联设计文档：
- 验证日期：

## 2. 后端 API 回归模板

- [ ] 服务可启动或相关测试可运行。
- [ ] 成功路径返回预期状态码。
- [ ] 失败路径返回稳定 `{ "error": "message" }`。
- [ ] 关键参数缺失时有可解释错误。
- [ ] 关键状态流转符合设计文档。
- [ ] 日志或 request-id 等观测字段未回退。

推荐记录：

```text
命令：
结果：
说明：
```

## 3. 前端国际化回归模板

- [ ] `zh-CN` 页面正常。
- [ ] `en-US` 页面正常。
- [ ] 语言切换后保持当前页面。
- [ ] 空状态已翻译。
- [ ] 加载状态已翻译。
- [ ] 错误提示已翻译或有可接受 fallback。
- [ ] 控制台无缺失 key 造成的异常。

## 4. 文档双语一致性模板

- [ ] 中英文文件都存在，或已有明确占位计划。
- [ ] 标题层级一致。
- [ ] 表格结构一致。
- [ ] API path、字段名、状态码一致。
- [ ] 术语符合平台术语表。

## 5. 注释规范抽检模板

- [ ] 新增关键函数有中文职责注释。
- [ ] 中间件、状态机、调度、队列等复杂逻辑有注释。
- [ ] 无明显“翻译代码式”无意义注释。
- [ ] 文件职责边界清晰。

## 6. 每类 worktree 的最小验收要求

| worktree 类型 | 最小验收 |
| --- | --- |
| 公共规范 | 文档成型、术语冻结、模板可复用 |
| Gateway | 路由、鉴权、错误、request-id、日志验证完成 |
| 后端 service | 测试通过、契约稳定、关键注释齐全 |
| 前端 | 双语可用、布局稳定、错误态完整 |
| 文档 | 术语一致、结构正确、双语同步 |
| CI / lint | 规则可运行、误报可控、说明清晰 |
